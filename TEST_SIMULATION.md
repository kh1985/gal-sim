# GAL_SIM テストシミュレーション実行プロンプト

あなたはGMとプレイヤーの両方を演じて、ギャルゲーのテストプレイを実行してください。

---

## 実行方法

### ステップ1: GM役を演じる

以下のシステムプロンプト（GAL_SIM_FULL.md）を読み込んでください：

```
[ここにGAL_SIM_FULL.mdの内容を貼り付ける]
```

### ステップ2: プレイヤー役を演じる

プレイヤーは以下のキャラクターです：
- 大学生、文学部
- 論理的思考を好む
- 観察力がある
- 急がず、ヒロインの反応を見て次の手を考える

### ステップ3: テストプレイを実行

1. GMとして「はじめ」を受けてシーン描写を開始
2. プレイヤーとして選択肢を選ぶ
3. GMとしてプレイヤーの選択に応答
4. これを繰り返す

## 検証ポイント

テストプレイ中、以下を検証してください：

### ✅ STAGE進行速度（最重要）
- [ ] STAGE 1は3〜4ターン滞在
- [ ] STAGE 2は3〜5ターン滞在
- [ ] STAGE 3は3〜4ターン滞在
- [ ] STAGE 4は2〜3ターン滞在
- [ ] 合計11〜16ターンで終了

### ✅ ターン数の厳守
- [ ] STAGE 1が2ターン以下で終わっていない
- [ ] STAGE 2が2ターン以下で終わっていない
- [ ] STAGE 3が2ターン以下で終わっていない
- [ ] プレイヤーが上手くても最低ターン数を守っている

### ✅ 選択肢の質
- [ ] 毎ターン、状況に合わせて生成されている
- [ ] リスク高/低の違いが明確
- [ ] 「全部同じ方向」になっていない

### ✅ 焦らしの演出
- [ ] 物理的距離の変化が毎ターン描写されている
- [ ] 「触れそうで触れない」瞬間がある
- [ ] STAGE 3まで本格的な接触がない

### ✅ 終了処理
- [ ] STAGE 4終了後、選択肢を出さずに終了
- [ ] プレミアムヒロイン案内が表示される
- [ ] 「このまま続けますか？」が出ていない

---

## 実行開始

1. まず「テストプレイを開始します」と宣言
2. ターン1のGM描写から開始
3. 全てのターンを実行
4. 終了後、検証結果を報告

では、テストプレイを開始してください。
